================================================================================
EXECUTIVE SUMMARY
================================================================================

Problem
-------
Current API documentation shows 100+ fragmented tags, making navigation
difficult for developers. Tags use inconsistent naming (kebab-case vs Title
Case), render controllers pollute the public API docs, and related endpoints
are scattered across categories.

Solution
--------
Reduce to 50 logically grouped, consistently named tags organized by developer
journey. Group related resources, separate administrative endpoints, and hide
internal render controllers.

Impact
------
✓ 50% reduction in tag count (100+ → 50)
✓ Improved navigation and discoverability
✓ Consistent Title Case naming throughout
✓ Related endpoints grouped and adjacent
✓ Admin endpoints clearly isolated
✓ Render controllers hidden from public API
✓ Scalable for future growth

================================================================================
KEY FEATURES OF THIS STRUCTURE
================================================================================

1. LOGICAL GROUPING
   - Tags organized by business domain and resource type
   - Related endpoints grouped together (Events → Calendars → Categories)
   - Clear hierarchical relationships

2. DEVELOPER JOURNEY
   - Ordered from most-used to least-used
   - Auth → Users → Content → Features → Admin
   - Follows typical API consumption pattern

3. CONSISTENCY
   - All tags use Title Case with Spaces (not kebab-case)
   - Admin tags prefixed with "Admin " for clarity
   - Resource names are specific and descriptive

4. ADMIN ISOLATION
   - 12 dedicated admin tags (79% reduction from current)
   - Grouped together at the end for easy filtering
   - Clear separation from public API resources

5. INTERNAL EXCLUSION
   - Render controllers marked as "internal"
   - HTML templates excluded from public API docs
   - Controllers affected:
     * jobs/controllers.py:JobRenderController
     * nominations/controllers.py:NominationRenderController
     * pages/controllers.py:PageRenderController
     * search/controllers.py:SearchRenderController
     * sponsors/controllers.py:SponsorRenderController
